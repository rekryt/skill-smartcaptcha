# Пример: Express + SmartCaptcha (фуллстек)

Читайте этот файл, когда нужен минимальный работающий образец «форма с капчей + серверная проверка токена» на Node.js, чтобы запустить его или перенести код в свой проект.

## Что где лежит

| Файл | Назначение |
| --- | --- |
| `server.js` | Express-сервер: отдает страницу с подстановкой sitekey, обрабатывает `POST /submit`, проверяет токен через `/validate` |
| `public/index.html` | Страница с формой и автоматическим виджетом (`div.smart-captcha`), плейсхолдер `__SITEKEY__` |
| `package.json` | Манифест; единственная зависимость — `express` |

## Как запустить

Требуется Node.js 18+ — пример использует глобальный `fetch` и `AbortSignal.timeout`, без сторонних HTTP-клиентов.

1. Установите зависимости:

   ```bash
   npm install
   ```

2. Задайте ключи капчи (где их взять — [../../docs/keys.md](../../docs/keys.md)):

   ```bash
   # bash
   export SMARTCAPTCHA_CLIENT_KEY="<ключ_клиента>"
   export SMARTCAPTCHA_SERVER_KEY="<ключ_сервера>"
   ```

   ```powershell
   # PowerShell
   $env:SMARTCAPTCHA_CLIENT_KEY = "<ключ_клиента>"
   $env:SMARTCAPTCHA_SERVER_KEY = "<ключ_сервера>"
   ```

3. Запустите сервер и откройте страницу:

   ```bash
   npm start
   # http://localhost:3000 (порт меняется переменной PORT)
   ```

## Как это работает

1. `GET /` — сервер читает `public/index.html` и подставляет ключ клиента вместо `__SITEKEY__`. Скрипт `https://smartcaptcha.yandexcloud.net/captcha.js` находит `div.smart-captcha` и отрисовывает виджет.
2. Пользователь проходит проверку — виджет добавляет в контейнер скрытое поле `<input type="hidden" name="smart-token" value="<токен>">`, и форма отправляет его на сервер.
3. `POST /submit` — сервер шлет `POST https://smartcaptcha.yandexcloud.net/validate` (`x-www-form-urlencoded`: `secret`, `token`, `ip`) и отвечает пользователю «Принято» или «Отклонено».

Ключевые решения (почему так):

* **Fail-open.** HTTP-код ответа не 200, таймаут или сетевая ошибка обрабатываются как успешная проверка — по рекомендации документации, чтобы сбой сервиса капчи не блокировал ваших пользователей и не задерживал ответ.
* **Только `status === "ok"`.** Решение принимается по полю `status`; поле `message` — только для диагностики в логах.
* **Токен одноразовый и живет 5 минут.** Повторная отправка той же формы вернет «Отклонено» (`Invalid or expired Token`) — это ожидаемо, пользователь должен пройти проверку заново.
* **`ip` передается всегда.** Параметр необязателен, но улучшает качество работы SmartCaptcha. За обратным прокси настройте `app.set('trust proxy', ...)`, иначе в `req.ip` будет IP прокси.

## Как проверить

* Кнопка **Я не робот** появляется на странице; задание показывается не всегда — откройте страницу в режиме инкогнито или через VPN, чтобы повысить вероятность задания.
* Если виджет не отображается — проверьте, что домен добавлен в список разрешенных сайтов в настройках капчи.
* Отправка формы без прохождения капчи возвращает 400 «пройдите проверку капчи».
