# Пример: Flask + SmartCaptcha (фуллстек)

Читайте этот файл, когда нужен минимальный работающий образец «форма с капчей + серверная проверка токена» на Python, чтобы запустить его или перенести код в свой проект.

## Что где лежит

| Файл | Назначение |
| --- | --- |
| `app.py` | Flask-сервер: отдает страницу с подстановкой sitekey, обрабатывает `POST /submit`, проверяет токен через `/validate` |
| `templates/index.html` | Jinja2-шаблон с формой и автоматическим виджетом (`div.smart-captcha`) |
| `requirements.txt` | Зависимости: `flask`, `requests` |

## Как запустить

1. Установите зависимости (лучше в виртуальном окружении):

   ```bash
   pip install -r requirements.txt
   ```

2. Задайте ключи капчи (где их взять — [../../docs/keys.md](../../docs/keys.md)):

   ```bash
   # bash
   export SMARTCAPTCHA_CLIENT_KEY="<ключ_клиента>"
   export SMARTCAPTCHA_SERVER_KEY="<ключ_сервера>"
   ```

   ```powershell
   # PowerShell
   $env:SMARTCAPTCHA_CLIENT_KEY = "<ключ_клиента>"
   $env:SMARTCAPTCHA_SERVER_KEY = "<ключ_сервера>"
   ```

3. Запустите сервер и откройте страницу:

   ```bash
   python app.py
   # http://127.0.0.1:5000
   ```

## Как это работает

1. `GET /` — Flask рендерит `templates/index.html`, подставляя ключ клиента в `data-sitekey="{{ sitekey }}"`. Скрипт `https://smartcaptcha.yandexcloud.net/captcha.js` находит `div.smart-captcha` и отрисовывает виджет.
2. Пользователь проходит проверку — виджет добавляет в контейнер скрытое поле `<input type="hidden" name="smart-token" value="<токен>">`, и форма отправляет его на сервер.
3. `POST /submit` — функция `check_captcha` (по образцу Python-примера из официального быстрого старта) шлет `POST https://smartcaptcha.yandexcloud.net/validate` (`x-www-form-urlencoded`: `secret`, `token`, `ip`) с `timeout=1` и отвечает пользователю «Принято» или «Отклонено».

Ключевые решения (почему так):

* **Fail-open.** HTTP-код ответа не 200, таймаут или сетевая ошибка (`requests.RequestException`) обрабатываются как успешная проверка — по рекомендации документации, чтобы сбой сервиса капчи не блокировал ваших пользователей и не задерживал ответ.
* **Только `status == "ok"`.** Решение принимается по полю `status`; поле `message` — только для диагностики в логах.
* **Токен одноразовый и живет 5 минут.** Повторная отправка той же формы вернет «Отклонено» (`Invalid or expired Token`) — это ожидаемо, пользователь должен пройти проверку заново.
* **`ip` передается всегда.** Параметр необязателен, но улучшает качество работы SmartCaptcha. Во Flask это `request.remote_addr`; за обратным прокси восстановите реальный IP (например, через werkzeug `ProxyFix`).

## Как проверить

* Кнопка **Я не робот** появляется на странице; задание показывается не всегда — откройте страницу в режиме инкогнито или через VPN, чтобы повысить вероятность задания.
* Если виджет не отображается — проверьте, что домен добавлен в список разрешенных сайтов в настройках капчи.
* Отправка формы без прохождения капчи возвращает 400 «пройдите проверку капчи».
