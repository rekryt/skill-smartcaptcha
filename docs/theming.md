# Тёмная и светлая тема виджета

Читай этот файл, когда виджет SmartCaptcha должен следовать теме сайта: «капча светлая при тёмной теме», «сделай тёмную капчу», «виджет выбивается из дизайна». Здесь два независимых рычага управления темой и готовые рецепты под оба типа сайтов — с системной темой и с ручным переключателем.

> **Источник знаний.** Управление темой в официальной документации описано только на уровне ресурса капчи («Динамическая цветовая схема», `style_json`). Всё про параметр `theme` и механику выбора темы внутри виджета — **реверс-инжиниринг `captcha.js` и HTML iframe чекбокса (июль 2026)**; это недокументированное поведение, оно может измениться без предупреждения. Как перепроверить актуальность — в конце файла.

## Два рычага

| Рычаг | Где настраивается | Что делает |
|---|---|---|
| **Наборы стилей** (светлый + тёмный) | Ресурс капчи в Yandex Cloud: «Внешний вид» → **«Динамическая цветовая схема»**, кастомизация — `style_json` с ключами `light`/`dark` ([captcha-management.md](captcha-management.md), [../references/management-api.md](../references/management-api.md)) | Даёт виджету тёмные токены стилей. Без этого тёмной темы у виджета нет вовсе |
| **Выбор активной темы** | Параметр `theme` в `render()` (недокументированный): `'light' \| 'dark' \| 'auto'` | Каким набором рисоваться прямо сейчас |

## Как это работает под капотом

Проверено по исходникам (минифицированный `captcha.js` + inline-скрипт `checkbox.<lang>.<hash>.html`):

1. `render()` принимает `theme` наравне с документированными параметрами (`sitekey`, `hl`, `test`, …) и передаёт его **в URL iframe виджета** (`?theme=...`).
2. Внутри iframe: `theme === 'dark'` → включается класс `Theme_dark` (тёмные токены); `theme === 'auto'` → `matchMedia('(prefers-color-scheme: dark)')` с подпиской на изменения; иначе — светлая.
3. **Сам `captcha.js` `prefers-color-scheme` в родительском окне не читает.** Тема страницы-родителя на виджет напрямую не влияет.

Следствия, из-за которых «просто включить Динамическую схему» часто не помогает:

- «Динамическая цветовая схема» без явного `theme` = режим `auto` → виджет следует **теме ОС/браузера** (`prefers-color-scheme` внутри iframe), а не теме сайта. Если у сайта ручной переключатель (класс `dark` на `<html>` — Tailwind, @nuxtjs/color-mode и т.п.), при светлой ОС капча останется светлой на тёмном сайте.
- Класс на `<html>` и CSS-переменные родителя в iframe не проникают. Проброс через CSS `color-scheme` на контейнере (спецификация CSS Color Adjustment: iframe выводит своё `prefers-color-scheme` из `color-scheme` элемента-владельца) браузерозависим — годится как подстраховка, но не как основной механизм.

## Рецепты

### Сайт следует теме ОС (нет своего переключателя)

Достаточно консоли: включи «Динамическую цветовую схему» у капчи. `theme` можно не передавать вовсе или передать `'auto'` явно. После включения опции сделай жёсткое обновление страницы — конфиг виджета кэшируется.

### Сайт с ручным переключателем темы

Передавай `theme: 'dark' | 'light'` из состояния темы сайта. Два обязательных момента:

- **`theme` фиксируется в URL iframe при `render()`** — на лету не меняется. При переключении темы сайта виджет пересоздаётся: снять подписки (`subscribe` возвращает функции отписки) → `destroy(widgetId)` → повторный `render()` с новым `theme`. Текущий токен при этом обнуляется — не страшно, он одноразовый.
- «Динамическую цветовую схему» в консоли **оставь включённой** — она даёт виджету сами тёмные стили.

```js
let widgetId = null;

function renderCaptcha() {
  const isDark = document.documentElement.classList.contains('dark');
  widgetId = window.smartCaptcha.render(container, {
    sitekey: SITEKEY,
    theme: isDark ? 'dark' : 'light', // недокументированный параметр — см. предупреждение выше
    callback: onToken,
  });
}

function onSiteThemeChange() {
  if (widgetId !== null) window.smartCaptcha.destroy(widgetId);
  renderCaptcha(); // токен обнулился — пользователь пройдёт проверку заново
}
```

Vue 3: composable [../templates/useSmartCaptcha.ts](../templates/useSmartCaptcha.ts) принимает `theme` в том числе как `Ref` и пересоздаёт виджет при смене значения сам — см. [vue.md](vue.md).

### Кастомные цвета тёмной темы

`style_json` с ключами `light` и `dark` (консоль → «Внешний вид» → вкладка JSON, CLI `--style-json`, Terraform `style_json`) — подгони токены под палитру сайта; предпросмотр в консоли ([captcha-settings.md](captcha-settings.md)).

## Диагностика

| Симптом | Причина | Решение |
|---|---|---|
| Капча светлая при тёмной теме сайта, ОС светлая | `auto` следует ОС, а не сайту | Явный `theme` из состояния темы сайта + пересоздание при переключении |
| Капча светлая всегда, даже при тёмной ОС | «Динамическая цветовая схема» выключена, или закэширован старый конфиг | Включить опцию в консоли; жёсткое обновление (Ctrl+F5) |
| Тема не меняется при переключении темы сайта | `theme` зафиксирован в URL iframe при `render()` | `destroy()` + повторный `render()` с новым значением |
| После `theme: 'dark'` виджет «прыгнул» по вёрстке | Пересоздание виджета заново меняет высоту контейнера | Контейнеру заранее `height: 100px` (как обычно) |

## Как перепроверить недокументированное поведение

При обновлении `captcha.js` (или если тема перестала работать):

```bash
curl -s "https://smartcaptcha.yandexcloud.net/captcha.js?render=onload" | grep -oE ".{80}theme.{40}" | head
# в выдаче должен остаться theme среди параметров render() и в URL-параметрах iframe;
# затем найти актуальный хэш страницы чекбокса и проверить обработку внутри iframe:
curl -s "https://smartcaptcha.yandexcloud.net/captcha.js" | grep -oE "checkbox[^\"']+html"
curl -s "https://smartcaptcha.yandexcloud.net/<имя_из_предыдущей_команды>" | grep -oE ".{60}theme=.{60}"
```

Ищи: `theme=window.location.search.match(...)` (чтение из URL) и `"dark"===theme?...Theme_dark...:"auto"===theme&&addThemeEventListener()` (логика выбора). Если этого больше нет — поведение изменилось, обнови этот файл.
