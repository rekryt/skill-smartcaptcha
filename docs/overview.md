# Как работает Yandex SmartCaptcha и как выбрать способ интеграции

Читайте этот файл первым: он объясняет общий принцип работы SmartCaptcha (виджет → токен → серверная проверка), свойства токена и помогает выбрать конкретный метод интеграции. Детали каждого метода — в отдельных файлах, на которые ссылается таблица ниже.

## Принцип работы

Yandex SmartCaptcha — сервис защиты от ботов и автоматизированных атак. Он проверяет запросы пользователей ML-алгоритмами и показывает задание только тем, чьи запросы посчитает подозрительными. Большинство легитимных пользователей задание не увидят вовсе.

Полный цикл проверки состоит из двух обязательных частей — клиентской и серверной:

1. **Виджет на странице.** Страница подключает скрипт `https://smartcaptcha.yandexcloud.net/captcha.js` и отрисовывает виджет по **ключу клиента** (публичный ключ капчи, см. [keys.md](keys.md)).
2. **Запуск проверки.** В обычной капче проверку запускает клик пользователя по кнопке **Я не робот**; в невидимой — вызов `window.smartCaptcha.execute()` в момент, который выбирает разработчик (например, на submit формы).
3. **Анализ и задание.** SmartCaptcha анализирует запрос. Если запрос выглядит подозрительным, пользователю показывается задание.
4. **Выдача токена.** По результату проверки сервис выдает **одноразовый токен**. В автоматическом методе токен попадает в скрытое поле внутри контейнера виджета:

   ```html
   <div id="captcha-container" class="smart-captcha" ...>
       <input type="hidden" name="smart-token" value="<токен>">
   </div>
   ```

   В расширенном методе токен приходит в колбэк `callback(token)`, в React — в обработчик `onSuccess(token)`.
5. **Отправка на бэкенд.** Фронтенд передает токен на ваш сервер — обычно вместе с данными формы.
6. **Серверная проверка.** Бэкенд отправляет POST-запрос на `https://smartcaptcha.yandexcloud.net/validate` с параметрами `secret` (**ключ сервера**), `token` и `ip` (в формате `x-www-form-urlencoded`) и по полю `status` в ответе (`ok` / `failed`) решает, пропускать ли пользователя. Подробности и готовый код — в [server-validation.md](server-validation.md).

> **Важно.** Токен сам по себе ничего не подтверждает — результат проверки известен только после запроса к `/validate` с ключом сервера. Проверка на клиенте невозможна и небезопасна: без шага 6 капча не защищает.

Сервис использует информацию об HTTP-запросах для развития своих ML-моделей; это можно отключить в консоли управления при создании капчи или позднее в ее настройках.

## Обычная и невидимая капча

* **Обычная капча** — виджет с кнопкой **Я не робот**. Клик по кнопке отправляет запрос пользователя в SmartCaptcha; задание показывается только при подозрительном запросе.
* **Невидимая капча** — кнопки на странице нет. Виджет отрисовывается с параметром `invisible: true` (только расширенным методом), а проверка запускается вызовом `window.smartCaptcha.execute()` в выбранный вами момент. Окно с заданием увидят только подозрительные пользователи. Невидимая капча требует меньше памяти (не загружает код кнопки), но вы обязаны уведомить пользователей об обработке их данных сервисом, если скрываете стандартный блок с уведомлением. Подробности — в [invisible-captcha.md](invisible-captcha.md).

## Выбор способа интеграции

Клиентская часть зависит от платформы; серверная проверка `/validate` одинакова и обязательна во всех сценариях.

| Сценарий | Метод | Файл скилла |
|---|---|---|
| Классический HTML-сайт, форма с кнопкой | Автоматический метод: `div` с классом `smart-captcha` | [widget-auto.md](widget-auto.md) |
| SPA, динамическая подгрузка виджета, нужны колбэки и управление жизненным циклом | Расширенный метод: `window.smartCaptcha.render()` | [widget-advanced.md](widget-advanced.md) |
| React-приложение | Официальный пакет `@yandex/smart-captcha` | [react.md](react.md) |
| Vue 3 / Nuxt | Composable на расширенном методе (официального пакета нет) | [vue.md](vue.md) |
| Капча без кнопки **Я не робот** | Невидимая капча: `render()` с `invisible: true` + `execute()` | [invisible-captcha.md](invisible-captcha.md) |
| Мобильное приложение (Android / iOS / Flutter) | HTML-страница с виджетом внутри WebView | [mobile-webview.md](mobile-webview.md) |
| Любой сценарий: проверка токена на бэкенде | POST `https://smartcaptcha.yandexcloud.net/validate` | [server-validation.md](server-validation.md) |

Почему так: автоматический метод — минимум кода, но виджет ставится один раз при загрузке страницы; расширенный метод дает контроль (момент отрисовки, колбэки, несколько виджетов, невидимый режим) и потому нужен для SPA и невидимой капчи. Полный список параметров и методов — в [../references/widget-params.md](../references/widget-params.md) и [../references/js-api.md](../references/js-api.md).

Создание самой капчи в Yandex Cloud (консоль, CLI `yc smartcaptcha captcha`, Terraform `yandex_smartcaptcha_captcha`, REST API) описано в [captcha-management.md](captcha-management.md); пошаговый сценарий — в [../workflows/setup-captcha.md](../workflows/setup-captcha.md).

## Жизненный цикл и свойства токена

Токен — это идентификатор проверенного запроса, который SmartCaptcha присваивает после проверки. Его свойства определяют архитектуру интеграции:

* **Срок жизни — 5 минут.** По истечении токен становится недействительным, и пользователю нужно снова пройти проверку. Не копите токены — отправляйте на бэкенд и валидируйте сразу после получения.
* **Одноразовый.** Токен можно использовать только один раз. Повторная попытка валидировать тот же токен вернет `"status": "failed"` с сообщением `Invalid or expired Token`. Поэтому не валидируйте один токен в нескольких местах (например, в middleware и в обработчике формы).
* **Результат — только через `/validate`.** Ответ сервиса — JSON с полями `status` (`ok` / `failed`) и `message`; при `status: ok` добавляется поле `host` — сайт, на котором пройдена проверка. Принимайте решение только по `status`; `message` — для диагностики, не для условий в коде.
* **Fail-open при сбое сервиса.** Если `/validate` вернул HTTP-код, отличный от 200, рекомендуется обрабатывать это как `"status": "ok"` — то есть пропустить пользователя. Иначе сбой или задержка SmartCaptcha заблокирует всех легитимных пользователей. Все варианты ответов — в [../references/validate-api.md](../references/validate-api.md).

## Валидация домена

При создании капчи указывается список сайтов, на которых она будет размещена. По умолчанию SmartCaptcha проверяет имя домена и позволяет использовать капчу только на сайтах из списка — это защищает ваш ключ клиента от использования на чужих сайтах. Проверку можно отключить (полезно, когда список доменов длинный, часто меняется или неизвестен), но тогда обязательно проверяйте поле `host` в ответе `/validate` на своем сервере и отклоняйте результаты с неожиданных доменов. Пустой `host` может означать блокировку облака или внутренний сбой сервиса — во втором случае его можно считать доверенным. Настройка списка доменов — в [captcha-settings.md](captcha-settings.md); если капча не отображается из-за домена — [troubleshooting.md](troubleshooting.md).

## Ограниченный режим

Если платежный аккаунт не в статусе `ACTIVE` или `TRIAL_ACTIVE`, SmartCaptcha работает в ограниченном режиме: все запросы на валидацию получают ответ `ok`, а виджет может показывать сообщение **Работа Yandex SmartCaptcha на странице ограничена**. Защита в этом режиме фактически не работает. Полнофункциональный режим восстанавливается автоматически при возврате аккаунта в статус `ACTIVE`/`TRIAL_ACTIVE`. Диагностика — в [troubleshooting.md](troubleshooting.md).
