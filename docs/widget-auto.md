# Автоматическое подключение виджета

Читайте этот файл, когда нужно добавить капчу на классическую HTML-страницу с формой, отправляемой обычным POST, — без JavaScript-логики на стороне клиента. Это самый быстрый способ интеграции: скрипт сам находит контейнеры и отрисовывает виджеты.

## Как это работает

1. Скрипт `captcha.js` загружается на страницу и после загрузки находит все элементы `div` с классом `smart-captcha`.
2. В каждом найденном контейнере отрисовывается виджет — кнопка **Я не робот**. Параметры виджета задаются data-атрибутами контейнера.
3. Пользователь нажимает кнопку; если запрос кажется сервису подозрительным, пользователю показывается задание.
4. После успешной проверки виджет добавляет внутрь контейнера скрытое поле `<input type="hidden" name="smart-token" value="<токен>">`.
5. Контейнер размещается внутри `<form>`, поэтому токен уходит на бэкенд вместе с остальными полями формы. Бэкенд обязан проверить токен запросом к `https://smartcaptcha.yandexcloud.net/validate` — без этого капча ничего не защищает (см. [server-validation.md](server-validation.md)).

Готовая страница целиком — в шаблоне [../templates/widget-auto.html](../templates/widget-auto.html).

## Шаг 1. Подключите скрипт

Разместите в любом месте страницы (например, внутри `<head>`):

```html
<script src="https://smartcaptcha.yandexcloud.net/captcha.js" defer></script>
```

Атрибут `defer` нужен, чтобы скрипт не блокировал отрисовку страницы и выполнился после построения DOM — к этому моменту контейнеры уже существуют и скрипт их найдет.

## Шаг 2. Добавьте контейнер в форму

Внутри `<form>` разместите пустой `div` с классом `smart-captcha`:

```html
<div
  id="captcha-container"
  class="smart-captcha"
  data-sitekey="<ключ_клиента>"
  style="height: 100px"
></div>
```

`<ключ_клиента>` — **Ключ клиента** со страницы капчи в консоли Yandex Cloud (вкладка **Обзор**). Как получить ключи и почему ключ сервера нельзя класть в HTML — в [keys.md](keys.md).

### Поддерживаемые data-атрибуты

| Data-атрибут    | Значение                                                                      | Значение по умолчанию       |
| --------------- | ----------------------------------------------------------------------------- | --------------------------- |
| `data-sitekey`  | `string` — ключ клиента                                                       | Отсутствует                 |
| `data-hl`       | `'ru'` \| `'en'` \| `'be'` \| `'kk'` \| `'tt'` \| `'uk'` \| `'uz'` \| `'tr'` — язык виджета | `window.navigator.language` |
| `data-callback` | `string` — имя глобальной функции, которая будет вызвана с токеном: `function callback(token) { ... }` | Отсутствует                 |

Это все параметры автоматического метода. Остальные возможности (`test`, `webview`, `invisible`, `shieldPosition`, `hideShield`) доступны только в расширенном методе через `window.smartCaptcha.render()` — полная таблица в [../references/widget-params.md](../references/widget-params.md).

`data-callback` полезен, например, чтобы держать кнопку отправки формы заблокированной до прохождения проверки. Для отправки токена на бэкенд колбэк не нужен — скрытое поле добавляется автоматически.

## Высота контейнера: задайте 100px заранее

> **Важно.** При загрузке виджет меняет высоту контейнера на `100px`. Если высота не была задана заранее, верстка страницы «прыгнет» в момент загрузки. Задайте контейнеру `style="height: 100px"` (или эквивалент в CSS) до загрузки виджета.

## Как токен уходит на бэкенд

После проверки токен появляется в контейнере как скрытое поле:

```html
<div id="captcha-container" class="smart-captcha" ...>
    <input type="hidden" name="smart-token" value="<токен>">
    ...
</div>
```

Так как контейнер лежит внутри `<form>`, при submit токен отправляется в теле POST-запроса под именем `smart-token`. На бэкенде прочитайте его из тела формы (например, `$_POST['smart-token']` в PHP или `request.form["smart-token"]` во Flask) и проверьте POST-запросом на `https://smartcaptcha.yandexcloud.net/validate` с ключом сервера. Токен одноразовый, срок жизни — 5 минут, поэтому проверяйте его сразу при обработке формы. Полная процедура, формат ответа и обработка ошибок — в [server-validation.md](server-validation.md), готовый код — в [../templates/validate.py](../templates/validate.py), [../templates/validate.js](../templates/validate.js), [../templates/validate.php](../templates/validate.php).

## Проверка работы

Кнопка **Я не робот** показывает задание не всегда — только при подозрении на бота. Чтобы увидеть задание при отладке, откройте страницу в режиме инкогнито или через VPN. Если виджет не отображается совсем — проверьте, что домен страницы добавлен в список разрешенных сайтов в настройках капчи (см. [troubleshooting.md](troubleshooting.md)).

## Когда автоматического метода мало

Переходите на расширенный метод ([widget-advanced.md](widget-advanced.md)), если нужно:

* управлять моментом отрисовки виджета (SPA, динамические формы) через `window.smartCaptcha.render()`;
* подписываться на события виджета (`success`, `token-expired`, `network-error` и другие) через `subscribe`;
* сбрасывать виджет (`reset`) без перезагрузки страницы или получать токен кодом (`getResponse`);
* включить невидимую капчу (`invisible`) — см. [invisible-captcha.md](invisible-captcha.md);
* включить тестовый режим (`test`) или режим WebView для мобильных приложений — см. [mobile-webview.md](mobile-webview.md).
